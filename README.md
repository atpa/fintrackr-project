# FinTrackr – Простое приложение для учёта личных финансов

Проект **FinTrackr** демонстрирует базовый пример веб‑приложения для ведения бюджета. Оно соответствует требованиям учебного задания: использует семантическую разметку HTML, адаптивные стили CSS, простой REST‑сервер на Node.js, хранит данные в JSON и показывает их с помощью удобного интерфейса. Приложение можно запустить локально и показать готовый функционал: добавленные счета, категории, операции и наглядные графики.

> **Обратите внимание:** В каталоге [`docs/`](docs/) находится подробное техническое задание по разработке личного кабинета пользователя, а также навигационная схема между разделами. Эти файлы помогут вам спланировать дальнейшее развитие проекта на уровне продукта **Revolut/Fintrackr** с элементами AI‑аналитики.

> **Новое в этой версии:** добавлена поддержка создания счётов, операций и бюджетов через веб‑формы. Теперь данные можно вносить и сразу видеть результат в таблицах и графиках. Изменения записываются в файл `data.json`, поэтому после перезапуска сервера внесённые данные сохраняются.

## Структура проекта

```
fintrackr/
├── backend/
│   ├── server.js        # Простенький HTTP‑сервер и реализация REST‑API
│   └── data.json        # Пример данных для тестирования (счета, категории, транзакции, бюджеты)
├── public/
│   ├── index.html       # Дэшборд с графиками и общей статистикой
│   ├── accounts.html    # Отдельная страница со счетами
│   ├── transactions.html# Страница списка операций
│   ├── budgets.html     # Страница с бюджетами
│   ├── css/
│   │   └── style.css    # Общие стили и адаптивная вёрстка
│   └── js/
│       ├── app.js       # Общие функции (загрузка данных, отображение графиков)
│       ├── accounts.js  # Логика страницы счётов
│       ├── transactions.js # Логика списка транзакций
│       └── budgets.js   # Логика страницы бюджетов
└── README.md            # Вы читаете этот файл
```

### `backend/server.js`
Запускает HTTP‑сервер на порту **3000**. Обрабатывает запросы к REST‑эндпоинтам:

- `GET /api/accounts` – список счетов.
- `GET /api/categories` – список категорий.
- `GET /api/transactions` – список операций.
- `GET /api/budgets` – список бюджетов.

Сервер также раздаёт статические файлы из каталога `public/` (HTML, CSS и JS). Данные подгружаются из `backend/data.json`.

### Фронтенд

Каждая HTML‑страница использует **семантические теги** (`<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>`). Стили прописаны в файле `public/css/style.css` с использованием современного CSS (flexbox, grid, переменные). Страницы адаптивны – на узких экранах элементы перестраиваются в одну колонку.

#### График расходов по категориям
На главном экране (`index.html`) отображается простой столбчатый график, построенный на `<canvas>`. Код для рисования находится в `public/js/app.js`. Он суммирует операции по категориям и визуализирует их.

### Как запустить

1. Убедитесь, что у вас установлен Node.js (версии ≥ 14). Node уже предустановлен в среде тестирования.
2. Перейдите в каталог `fintrackr/backend` и запустите сервер:

   ```bash
   node server.js
   ```

3. Откройте браузер и перейдите по адресу [http://localhost:3000](http://localhost:3000). Вы увидите дэшборд.
4. Навигация между страницами осуществляется по ссылкам в верхнем меню (`Счета`, `Операции`, `Бюджеты`).

## Важные замечания

- Проект демонстрационный и не предусматривает полноценную авторизацию или сохранение данных между перезапусками сервера.
- Для простоты данные хранятся в JSON‑файле; в реальном приложении следует использовать базу данных (например, PostgreSQL).
- График рисуется простыми средствами Canvas, без внешних библиотек.

Проект можно развивать: добавить формы для добавления операций, поддержку мультивалютности, аутентификацию пользователей и интеграцию с внешними API (курсы валют, банковские выписки).
