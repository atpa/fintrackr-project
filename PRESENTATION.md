# Презентация проекта: FinTrackr

---

**Слайд 2: Проблема и Целесообразность (Критерий 1)**

-   **Проблема:** Исходный проект имел устаревшую архитектуру, проблемы с безопасностью и неоптимальный пользовательский опыт. Это делало его трудным в поддержке и неконкурентоспособным.
-   **Цель рефакторинга:** Создать надежную, быструю и безопасную платформу, готовую к дальнейшему развитию.
-   **Экономическое обоснование:** Снижение затрат на поддержку, повышение привлекательности для пользователей, создание базы для будущей монетизации (например, премиум-функции).

---

**Слайд 3: Архитектура и Технологии (Критерий 9)**

-   **Backend:**
    -   **Node.js + Express.js:** Надежная и быстрая основа для REST API.
    -   **Слоёная архитектура (Routes → Controllers → Services → Repositories):** Четкое разделение ответственности, упрощает тестирование и поддержку.
    -   **Аутентификация:** JWT (JSON Web Tokens), передаваемые через безопасные `HttpOnly` cookie.
    -   **Безопасность:** Использование `helmet` для защиты от веб-уязвимостей и `cors` для контроля доступа.

-   **Frontend:**
    -   **Vanilla JavaScript (SPA):** Быстрое и отзывчивое одностраничное приложение без тяжелых фреймворков.
    -   **Vite:** Современный и быстрый сборщик проектов.
    -   **Компонентный подход:** UI разбит на переиспользуемые компоненты (Card, Button, Input).
    -   **Дизайн-система:** Современный UI/UX в стиле FinTech-приложений (Revolut/Monzo) на основе CSS Variables.

---

**Слайд 4: Демонстрация Функциональности (Критерий 2, 5, 7)**

-   **Аутентификация:** Безопасный вход и регистрация.
-   **Панель управления (Dashboard):** Обзор финансового состояния.
-   **Управление счетами:** Просмотр и добавление счетов.
-   **История транзакций:** Таблица со всеми операциями.
-   **Финансовые отчеты:**
    -   Диаграмма расходов по категориям.
    -   График "Доходы vs Расходы".
-   **Комфортная навигация:** Логичная структура и удобная боковая панель.

---

**Слайд 5: Качество кода и Дизайн (Критерий 3, 6)**

-   **Семантическая HTML-разметка:** Использование тегов `<main>`, `<aside>`, `<nav>`, `<header>` для улучшения доступности и SEO.
-   **Современный CSS:** Применение Flexbox, Grid, CSS-переменных.
-   **Адаптивный дизайн:** Корректное отображение на всех устройствах, от мобильных телефонов до настольных компьютеров. Реализовано выезжающее мобильное меню.
-   **Кроссбраузерность:** Обеспечивается использованием стандартных веб-технологий и сборщиком Vite.

**Слайд 6: Тестирование и Контроль версий (Критерий 4)**

-   **Система контроля версий:** Проект велся в Git, все изменения фиксировались через осмысленные коммиты, отражающие этапы рефакторинга.
-   **Тестирование Backend:**
    -   Используется фреймворк **Jest** и библиотека **Supertest**.
    -   Написаны тесты для проверки аутентификации и основных эндпоинтов API.
    -   Тесты гарантируют, что ключевая логика бэкенда работает корректно.
    -   Запуск тестов командой `npm test`.
