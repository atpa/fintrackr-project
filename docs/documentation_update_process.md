# Регулярное обновление документации FinTrackr

Документ описывает регламент поддержки документации после изменений в кодовой базе.

## 1. Ответственность

- **Автор изменений** — обновляет затронутые документы и фиксирует правки в PR.
- **Ревьюер** — проверяет соответствие документации реализованному функционалу и дорожной карте.
- **Техлид** — раз в месяц проводит аудит документов и планирует корректировки roadmap.

## 2. Триггеры для обновления

Документацию необходимо обновлять, если выполняется хотя бы одно из условий:

1. Добавлен или удалён функционал фронтенда или бекенда.
2. Изменена архитектура или схема данных.
3. Появились новые зависимости, интеграции, ограничения или риски.
4. Изменился roadmap, сроки или ответственные.

## 3. Процесс при создании PR

1. Обновить README, профильные спецификации и схемы.
2. Запустить проверку актуальности документации:
   ```bash
   npm run docs:status
   ```
3. Приложить в описании PR ссылку на обновлённые разделы и кратко описать изменения.
4. Получить ревью и убедиться, что контрольный список документации отмечен.

## 4. Ежемесячный цикл

1. В первую неделю месяца техлид запускает аудит:
   - просматривает `docs/architecture_overview.md`, `docs/personal_cabinet_spec.md`, `README.md`;
   - сравнивает с реальным состоянием системы;
   - создаёт задачи на корректировку, если выявлены расхождения.
2. Команда обновляет roadmap и фиксирует статус выполнения задач по документации.

## 5. Инструменты поддержки

- `npm run docs:status` — выводит список файлов документации, изменённых относительно `main`.
- Таблица в таск‑трекере с контрольными датами и ответственными.
- Хранилище диаграмм (Draw.io/Excalidraw) с версионированием.

## 6. Чек‑лист перед релизом

- [ ] README отражает текущие возможности и ограничения.
- [ ] Спецификация обновлена и синхронизирована с roadmap.
- [ ] Архитектурная схема актуальна и подписана датой обновления.
- [ ] Документация выложена в Confluence/Notion (при наличии) и синхронизирована с репозиторием.
- [ ] В changelog добавлена секция «Документация».

> Документ обновляется при каждом изменении регламента или инструмента сопровождения.
